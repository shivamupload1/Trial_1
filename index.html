Drive workflow

  1. Admin connects one Google Drive account for each client.
  2. Admin panel creates client and event.
  3. Backend creates event and album folders in that client's Drive.
  4. Uploads save drive_file_id and metadata in the database.
  5. Client sees watermarked previews and thumbnails.
  6. Download requests produce temporary controlled links.

Fast website rules

  • Website loads thumbnails first, not original 40MB photos.
  • Videos stay in Google Drive and open only through controlled links.
  • Images use lazy loading so lower sections load after scroll.
  • Database stores file IDs and status, not heavy media bytes.
